Concurrency TS Editor's Report, May 2014

Overview

N3970 is the Concurrency TS Draft. It is based on D3904 with the following changes as per LEWG feedback:

  • The then method now implements auto-unwrapping, obviating the need for unwrap.
  • The type thread_executor renamed to thread_per_task_executor.
  • Fixed an issue where is_ready was incorrectly spelled as ready in some parts of the document.
  • The method when_any_swapped has been renamed to when_any_back.

The new document also incorporates feedback from the following individals:

  • Agustín K-ballo Bergé pointed out several instances where reference decay was handled incorrectly, as well as numerous typos.
  • Vicente J. Botet Escriba found that it is underspecified how an exception thrown from a continuation is propagated into the returned future.

Additionally, the document has been fully reformatted and many formatting and editorial issues fixed.

The full history of changes can be traced at https://github.com/cplusplus/concurrency_ts.
